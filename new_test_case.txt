# 信用卡功能测试案例（成功与失败）

## 1. 信用卡还款成功
**内容:**
1. 用户登录系统，进入信用卡还款页面。
2. 输入还款金额500元，选择还款账户余额充足。
3. 提交还款请求，系统校验账户余额充足，扣款成功。
4. 信用卡可用额度增加500元，还款账户余额减少500元。
5. 系统返回还款成功提示。

## 2. 信用卡还款失败-还款账户余额不足
**内容:**
1. 用户登录系统，进入信用卡还款页面。
2. 输入还款金额2000元，选择还款账户余额仅1000元。
3. 提交还款请求，系统校验账户余额不足，拒绝还款。
4. 信用卡可用额度不变，还款账户余额不变。
5. 系统返回还款失败提示，原因：余额不足。

## 3. 信用卡挂失成功
**内容:**
1. 用户发现信用卡遗失，登录系统申请挂失。
2. 系统校验卡状态为正常，允许挂失。
3. 用户确认挂失，系统将卡状态更新为“挂失”。
4. 系统返回挂失成功提示，卡片无法再进行交易。

## 4. 信用卡挂失失败-卡状态异常
**内容:**
1. 用户尝试对已注销的信用卡进行挂失操作。
2. 系统校验卡状态为“注销”，不允许挂失。
3. 系统返回挂失失败提示，原因：卡状态异常。

## 5. 信用卡消费交易失败-卡片已过期
**内容:**
1. 用户使用已过期的信用卡进行消费交易。
2. 系统校验卡片有效期，发现已过期。
3. 系统拒绝交易，返回授权失败响应码。
4. 信用卡账户可用额度不变。

## 6. 信用卡调额成功
**内容:**
1. 用户申请将信用卡额度从10000元提升至20000元。
2. 系统进行审批，审批通过。
3. 信用卡额度更新为20000元，可用额度同步调整。
4. 系统返回额度调整成功提示。

## 7. 信用卡调额失败-审批未通过
**内容:**
1. 用户申请将信用卡额度从10000元提升至50000元。
2. 系统进行审批，因用户信用评分不足，审批未通过。
3. 信用卡额度保持10000元不变。
4. 系统返回额度调整失败提示，原因：审批
